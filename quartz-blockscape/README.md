# quartz-blockscape

Quartz v5 transformer plugin for rendering Blockscape `.bs` maps inside Quartz pages.

The v1 plugin supports:

- inline fenced maps using `blockscape` or `bs`
- source embeds using `<div class="blockscape-root" data-src="./maps/example.bs"></div>`
- lite and full renderers
- series rendering from JSON arrays
- build-time validation for inline maps
- read-only browser renderers

`.bs` files remain the canonical map source. The inline form is useful for small examples and documentation snippets.

## Install

Quartz plugins are usually installed from a standalone git package that ships `dist/`. This repository is a monorepo, so installing directly from the `quartz-blockscape/` subdirectory may not work with every Quartz package workflow.

For local development from this checkout:

```sh
cd quartz-blockscape
npm install
npm run build
```

Then link or copy this package into your Quartz site while testing.

The clean publishing path is to extract `quartz-blockscape/` into its own repository, commit `dist/`, and install that repository from your Quartz site. In this monorepo, `dist/` is generated by `npm run build`; publishable standalone releases should include it.

## Quartz Configuration

In your Quartz config, import and add the transformer:

```ts
import { Blockscape } from "quartz-blockscape"

export default {
  plugins: {
    transformers: [
      Blockscape({
        defaultRenderer: "lite",
      }),
    ],
  },
}
```

The plugin injects its client runtimes and CSS as Quartz external resources when `dist/` is available. `lite` is the backward-compatible default renderer. Use `defaultRenderer: "full"` to make unmarked embeds use the full Blockscape UI.

## Renderers

`lite` is the small read-only renderer from the first Quartz integration.

`full` embeds the main Blockscape Svelte UI with the top menu, side menu, footer, file open/save, local backend, and editing affordances disabled. Full mode keeps the richer map interaction surface, including series/version navigation.

## Inline Usage

````markdown
```blockscape full
{
  "id": "example",
  "title": "Example",
  "abstract": "Optional short description",
  "categories": [
    {
      "id": "outcomes",
      "title": "Outcomes",
      "items": [
        {
          "id": "understanding",
          "name": "Understanding",
          "deps": ["renderer"]
        }
      ]
    },
    {
      "id": "platform",
      "title": "Platform",
      "items": [
        {
          "id": "renderer",
          "name": "Renderer"
        }
      ]
    }
  ]
}
```
````

The `bs` fence alias is also supported.

Omit `full` to use the configured default renderer. With the default config, that means `lite`.

## Source Embed Usage

Keep the map in a `.bs` file under your Quartz content tree:

```html
<div class="blockscape-root" data-src="./maps/skupper.bs"></div>
```

Use the full renderer explicitly:

```html
<div class="blockscape-root" data-src="./maps/skupper.bs" data-blockscape-renderer="full"></div>
```

The client fetches the map relative to the rendered page URL. The transformer does not rewrite this marker in v1.

## Series Usage

A series is a JSON array of map objects in a fenced block or `.bs` file:

````markdown
```blockscape full
[
  {
    "id": "example-v1",
    "title": "Example v1",
    "categories": [
      {
        "id": "outcomes",
        "title": "Outcomes",
        "items": [{ "id": "understanding", "name": "Understanding" }]
      }
    ]
  },
  {
    "id": "example-v2",
    "title": "Example v2",
    "categories": [
      {
        "id": "outcomes",
        "title": "Outcomes",
        "items": [{ "id": "understanding", "name": "Understanding" }]
      }
    ]
  }
]
```
````

For larger series, prefer a `.bs` file and a `data-src` embed with `data-blockscape-renderer="full"`.

## Validation

Inline maps and inline series arrays are validated during the Quartz Markdown transform. The validator checks each map for:

- root value is an object, or a non-empty array of map objects for a series
- root `id`
- root `title`
- root `categories` array
- category `id`, `title`, and `items`
- item `id` and `name`

The plugin intentionally does not normalize or rewrite the map.

## Out Of Scope For V1

The integration remains a transformer plugin only. Full `.bs` page-type support, editor UI, Quartz graph synchronization, Obsidian canvas conversion, and server-side SVG rendering are intentionally left for later versions.

## Troubleshooting

Invalid JSON: the Quartz build should fail with `Invalid Blockscape JSON in <path>`. Fix the fenced JSON or move the canonical map into a `.bs` file and use `data-src`.

Invalid series: arrays must contain at least one valid map object. The build error points to the failing `series[index]` entry when possible.

Missing map file: check that the `.bs` file is copied by Quartz as a static asset and that the `data-src` path is relative to the rendered page.

Client-side fetch failure: open the browser console and look for `[Blockscape] fetching map` and `[Blockscape] failed to render map`. A 404 usually means Quartz did not publish the `.bs` file at the expected URL.

Maps not rendering after SPA navigation: the runtime listens for `nav`, `quartz:render`, and `popstate`, and also exposes `window.BlockscapeQuartz.mount()`. Call that after custom navigation if your Quartz setup emits a different event.

CSS/layout width problems: the renderer uses horizontal overflow for wide maps. Ensure parent containers do not set `overflow: hidden` or a very narrow fixed width.

Full renderer layout problems: full mode needs more vertical space than lite mode. Override `.blockscape-root--full { min-height: ... }` in your Quartz CSS if the map is cramped.

Bundle size: full mode ships the Svelte Blockscape app bundle in addition to the lite renderer. Use lite mode for pages that only need compact static map embeds.
